## What Speaksense Does

Speaksense is an AI platform for YouTube agencies — a conversational co-agent that replaces manual channel audits, competitor reports, and weekly reporting. Agencies query their entire client portfolio in natural language and get structured, data-backed answers in seconds.

Built from zero as Founder & AI Engineer: product, architecture, data model, AI layer, and billing.

## Co-Agent Architecture

The core is an English-to-SQL co-agent with parallel tool-calling and streaming inference via Vercel AI SDK v5.

**Why English-to-SQL over pre-built queries**: YouTube agency analytics are unbounded in what they need to ask — client comparisons, competitor gaps, subscriber trends, thumbnail performance. Pre-built queries can't cover that surface. The co-agent generates SQL at query time against the live database, validated before execution.

**Parallel tool-calling**: Multiple tools can fire concurrently within a single response — e.g., fetching channel stats while simultaneously running competitor analysis. This reduces latency and makes the co-agent feel responsive rather than sequential.

**8 registered tools**:

| Tool | Function |
|------|----------|
| `getChannelInfo` | Channel data + subscriber trends |
| `runSqlQuery` | NL → validated SQL → live results |
| `buildFilters` | Dynamic multi-channel filtering |
| `getCompetitorInfo` | Competitor gap analysis |
| `getThumbnailSummary` | AI thumbnail evaluation |
| `getVideoSummary` | Content performance synthesis |
| `createImage` | AI thumbnail generation |
| `getRedditContext` | Community trend signals |

## Data Layer

35+ PostgreSQL tables on Neon, structured for multi-tenant agency isolation:

- **Agency scoping**: Every query filters by `agency_id`. Cross-agency data exposure is architecturally impossible — not just policy.
- **RBAC via JSONB**: `agency_users.permissions` stores channel, user, billing, and analytics rights as a JSONB object. Role hierarchy: owner → admin → agent.
- **pgvector semantic search**: Video transcripts and content embedded for similarity search — enables queries like "find videos similar to our best performer."
- **Stripe + webhook state**: Plan changes propagate via webhooks to `user_plans`. Subscription state is always consistent with billing events.

## Platform

- **Next.js 15 App Router**: Server components for data fetching, client components for chat UI, server actions for mutations.
- **Stack Auth**: JWT-based authentication with per-tenant session isolation.
- **Persistent chat memory**: Sessions persist across logins with durable `user_chat_memory` and `memory_summaries` — the co-agent remembers context across conversations.
- **Production on Vercel**: Edge-deployed, streaming responses from first token.
